# collision-zone-thinking — concept collision thinking

Force two unrelated concepts together to discover emergent properties. "What if we treated X like Y?"

## Core principle

Revolutionary insights come from deliberately mixing metaphors. Do not iterate within the old domain — import a concept from a different domain.

## When to use

| Symptom | Action |
|---|---|
| Already tried every direction in this domain | Force a collision with an outside domain |
| Solutions feel incremental, no breakthrough | Need new ideas, not optimization of the old |
| "I've tried everything" | Import a concept from elsewhere |
| Need innovation, not optimization | Mix unrelated metaphors |

## Process

1. **Choose two unrelated concepts** from different domains
2. **Force the combination** — "What if we treated [A] like [B]?"
3. **Explore emergent properties** — what capabilities appear?
4. **Test the boundaries** — where does the metaphor break?
5. **Extract the insight** — what was learned?

## Rich source domains

- **Physics** — forces, thermodynamics, relativity
- **Biology** — evolution, ecosystems, immune systems
- **Economics** — markets, incentives, game theory
- **Architecture** — structure, flow, space utilization
- **Electrical engineering** — circuit breakers, fuses, load balancing

## Example

**Problem:** Distributed system with cascading failures
**Collision:** "What if we treated a service like an electrical circuit?"
**Emergent properties:** Circuit breaker, fuse, ground fault isolation, voltage regulation
**Where it breaks:** Circuits have no retry logic or self-healing
**Insight:** Failure isolation patterns from electrical engineering — apply to service mesh

## Notes

- Wilder combinations -> more surprising insights
- Test metaphor boundaries seriously
- Record failed collisions too (they also teach)
- Breakthrough > incremental improvement
