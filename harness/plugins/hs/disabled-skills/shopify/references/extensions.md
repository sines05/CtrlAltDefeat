# Extensions Reference

Guide for building Shopify UI extensions and Shopify Functions with current CLI-generated project structure.

Extension config varies by extension type and Shopify CLI template. Prefer generating with Shopify CLI, then edit the generated files instead of copying a universal TOML shape.

> **UI extensions use Preact + web components** via the global `shopify` object. The React model (`@shopify/ui-extensions-react/*`, `reactExtension()`, hooks like `useApi()`) is deprecated. Checkout, customer-account, and POS extensions must migrate by **2026-10-01** (API 2025-07 is the last React-compatible version); admin extensions also use Preact + Polaris web components, though a React
> retirement date is not yet published. The extension target is declared only in TOML (`[[extensions.targeting]]`), not passed in JS. Verify exact component element/attribute names against your generated template and the shopify.dev component reference.

## Checkout UI Extensions

Customize checkout and thank-you pages with native-rendered components.

### Common Targets

- `purchase.checkout.block.render` - merchant-configurable checkout block
- `purchase.thank-you.block.render` - thank-you page block
- `purchase.checkout.header.render-after`
- `purchase.checkout.contact.render-before`
- `purchase.checkout.shipping-option-list.render-after`
- `purchase.checkout.payment-method-list.render-after`
- `purchase.checkout.footer.render-before`

### Setup

```bash
# Use the interactive picker and select the checkout UI extension type.
shopify app generate extension
```

The legacy `--type checkout_ui_extension` flag no longer exists. Use the interactive picker, or verify any non-interactive `--template` value against your current Shopify CLI version.

Representative `shopify.extension.toml` shape:

```toml
api_version = "2026-04"

[[extensions]]
type = "ui_extension"
name = "gift-message"
handle = "gift-message"

[[extensions.targeting]]
target = "purchase.checkout.block.render"
module = "./src/Checkout.jsx"

[extensions.capabilities]
network_access = true
api_access = true
```

Always verify the exact fields against the file generated by your current Shopify CLI version and target.

### Basic Preact Example

Render Preact into the extension root and use web-component elements (`ui-*`). The target lives in TOML, not in JS.

```javascript
import { render } from 'preact';

export default function () {
  render(<Extension />, document.body);
}

function Extension() {
  return (
    <ui-section>
      <ui-heading>Order options</ui-heading>
      <ui-paragraph>Customize this order at checkout.</ui-paragraph>
      <ui-button onClick={() => shopify.action?.()}>Continue</ui-button>
    </ui-section>
  );
}
```

Use the verified primitives (`ui-section`, `ui-heading`, `ui-paragraph`, `ui-text`, `ui-button`) as a starting point. For inputs (text fields, checkboxes) and interactions like applying cart attributes, verify the exact element names and APIs against your generated template and the shopify.dev component reference — do not assume element/attribute names.

## Admin UI Extensions

Admin extensions render with Preact + Polaris web components (`s-admin-block`, `s-box`, `s-text`, etc.), not React. Generate with the interactive picker and select the admin action or admin block type:

```bash
shopify app generate extension
```

Verify the exact `--template` value (for example, admin action or admin block templates) against your current Shopify CLI version rather than hardcoding it.

```javascript
import { render } from 'preact';

export default function () {
  render(<Extension />, document.body);
}

function Extension() {
  return (
    <s-admin-block>
      <s-text>Product details</s-text>
    </s-admin-block>
  );
}
```

Example targets:
- `admin.product-details.action.render`
- `admin.order-details.action.render`
- `admin.customer-details.action.render`
- `admin.product-details.block.render`
- `admin.order-details.block.render`
- `admin.customer-details.block.render`

Representative Admin action config:

```toml
api_version = "2026-04"

[[extensions]]
type = "admin_action"
name = "Export product"
handle = "export-product"

[[extensions.targeting]]
target = "admin.product-details.action.render"
module = "./src/Action.jsx"
```

If an Admin extension calls your backend, pass and verify Shopify session tokens server-side. Do not expose Admin API access tokens in extension code.

## POS UI Extensions

```bash
# Use the interactive picker and select the POS UI extension type.
shopify app generate extension
```

POS extensions use Preact + web components (`s-tile`, etc.) and the global `shopify` action API. Common targets include smart grid tiles and modal/screen flows. Verify target and element names against generated CLI templates for your POS extension type.

```javascript
import { render } from 'preact';

export default function () {
  render(<Tile />, document.body);
}

function Tile() {
  return (
    <s-tile
      title="Gift Cards"
      subtitle="Manage gift cards"
      onClick={() => shopify.action.presentModal()}
    />
  );
}
```

## Customer Account Extensions

```bash
# Use the interactive picker and select the customer account UI extension type.
shopify app generate extension
```

Customer-account extensions use Preact + web components (`s-*`); read the signed-in customer via `shopify.authenticatedAccount`.

Representative targets:
- `customer-account.order-status.block.render`
- `customer-account.order-index.block.render`
- `customer-account.profile.block.render`

```javascript
import { render } from 'preact';

export default function () {
  render(<Extension />, document.body);
}

function Extension() {
  return (
    <s-box>
      <s-text>Need to return an item?</s-text>
      <s-button onClick={() => {}}>Start return</s-button>
    </s-box>
  );
}
```
